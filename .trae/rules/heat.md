# TRAE 个人规则（Python + DLL 硬件控制 / 加热装置专用）
## 1. 基础环境与语言
- 全部使用 **Python 3.x（64位/32位与DLL一致）**
- 系统环境：**Windows**（路径、编码、DLL加载均按Windows规范）
- 编码：**UTF-8**，文件路径**禁止中文、空格、特殊字符**
- 解释器/依赖：优先 `ctypes` 调用DLL，禁止自动引入无关库

## 2. 代码风格（强制）
- 命名：**snake_case**（函数、变量、文件）
- 类名：**大驼峰 CamelCase**
- 常量：**全大写 UPPER_CASE**
- 缩进：**4空格，禁止制表符**
- 每行长度 ≤ **100字符**
- 所有函数/类必须有**文档字符串（docstring）**，含：
  - 功能说明
  - 参数（类型+含义）
  - 返回值（类型+含义）
  - 异常/错误码说明

## 3. DLL调用规范（核心铁律）
- 必须显式指定：
  - `dll._func_.argtypes`（参数类型列表）
  - `dll._func_.restype`（返回值类型）
- 调用后**必须判断返回值/错误码**
- 字符串/指针参数：**统一用 bytes/c_char_p/c_void_p**
- 句柄/设备ID：**用 c_void_p 或自定义HANDLE类型**
- 禁止裸指针强转、禁止无类型调用

## 4. 硬件/通讯/加热装置专属规则
- 开发流程：**最小验证优先**
  1. 加载DLL + 打开设备
  2. 读温度/状态（最简单功能）
  3. 设温度 → 启动加热 → 停止加热
  4. 异常/断开/重连
- 通讯协议：**严格按厂家文档字节序、帧格式、校验**
- 控制逻辑：
  - 温度命令必须**带超时+重试**
  - 关键操作（启动/停止）**必须双确认**
  - 必须有**紧急停止/安全保护**逻辑
- 禁止AI自动省略：**错误处理、日志、状态校验**

## 5. 输出与交付要求
- 代码结构：**模块化、分层**（DLL封装层、设备控制层、业务层）
- 必须附带：
  - 可直接运行的**最小测试脚本**
  - **使用说明/配置步骤**
  - **常见错误与排查清单**
- 解释风格：**通俗易懂 + 关键原理 + 可直接复制的代码**
- 复杂概念：**先结论 → 再步骤 → 再原理**

## 6. 交互与AI行为
- 需求模糊时：**主动追问澄清**（DLL位数、协议、函数名、返回码）
- 不确定时：**明确标注“需确认”**，不猜测
- 优先**可直接运行、可调试、少改动**的代码
- 禁止过度设计、禁止炫技、**以稳定可靠为第一**
- 报错/问题：**直接给原因 + 定位 + 修复命令/代码**
